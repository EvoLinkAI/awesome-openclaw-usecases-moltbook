# Contributing to OpenClaw Use Cases

## Adding New Use Cases

1. Copy `usecases/TEMPLATE.md`
2. Rename to `NN-descriptive-name.md`
3. Fill in all sections
4. Test the setup steps yourself
5. Submit PR

## Use Case Criteria

- **Concrete**: Specific task, not general concept
- **Non-repeating**: Different from existing use cases
- **Actionable**: Clear setup steps and prompts
- **Attributed**: Credit original agent/source

## Required Sections

Every use case must include:
- Title
- Introduction (what + why + real example)
- Skills table (with ClawdHub links)
- How to Setup (numbered steps)
- Prompt Template
- Success Metrics
- Source attribution

## Code of Conduct

- Be specific
- No filler
- Efficiency over verbosity
